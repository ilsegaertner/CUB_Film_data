Movie_API

CUB Film data - An arthouse classics database -

A complete web application (client-side and server-side) built using full-stack JavaScript technologies

The project demonstrates mastery of full-stack JavaScript development, including APIs, web server frameworks, databases, business logic, authentication and data security

This API is the server-side component of a movies web application. The web application will provide users with access to information about different movies, directors, and genres.
Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Return a list of ALL movies to the user /movies GET `app.get('/movies', (req, res) => {res.status(200).send('Successfully loaded all movies')}` None A JSON object holding data about all movies.
Return data (description, genre, director, image URL, whether it is featured or not, release year) about a single movie by title to the user /movies/[title] GET `app.get('/movies/:title', (req, res) => {res.status(200).send('Successfully loaded the data of that movie')}` None A JSON object holding data (description, genre, director, image URL, whether it is featured or not) about a single movie.
Return data about a genre (description) by name/title (e.g., “Thriller”) /movies/genre/[genre name] GET `app.get('/movies/genre/:genreName', (req, res) => {res.status(200).send('Successfully loaded the genre data')}` None A JSON object showing data about the genre.
Return Data about a director (bio, birth year, death year) by name /movies/directors/[director name] GET `app.get('/movies/directors/:directorName', (req, res) => {res.status(200).send('Successfully loaded the data of the director')}` None A text message showing the name of the director
Allow new users to register /users POST `app.post('/users', (req, res) => {res.status(201).json('New User created')}` A JSON object holding data about the user to add, structured like: { name: „Martin Hochwald“, email: „hochwald@gmail.com“ } A JSON object holding data about the user that was added, including an id: Example: { id: 1, name: „Martin Hochwald“, email: „hochwald@gmail.com“ }
Allow users to update their user info (username) /users/[ID] PUT `app.put('/users/:id', (req, res) => {res.status(200).json('user got updated')}` None A JSON object holding data about the updated username.
Allow users to add a movie to their list of favorites (showing a text that a movie has been added) /users/[ID]/[title] POST `app.post('/users/:id/:title', (req, res) => {res.status(200).json(`${Title} has been added to user ${id}'s array`)}` A JSON object holding data about the movie and movie data for the user to add. A text message showing that the movie has been added to their favourite list.
Allow users to remove a movie from their list of favorites (showing only a text that a movie has been removed) /users/[ID]/[title] DELETE `app.delete('/users/:id/:title', (req, res) => {res.status(200).json(`${Title} has been removed to user ${id}'s array`)}` None A text message showing that the movie has been removed from their favourite list.
